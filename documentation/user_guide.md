## 1. Introduction

The DataCycle platform provides end-to-end analytics for the **HES-SO Bellevue campus** in Sierre, Switzerland. It collects, transforms, and visualises data from:

- **Solar energy production** — 5 photovoltaic inverters on-site
- **Building energy consumption** — real-time power draw across the campus
- **Room bookings** — occupancy and scheduling information
- **Environmental conditions** — indoor temperature, humidity, and weather forecasts

Data flows daily through a three-tier pipeline (Bronze → Silver → Gold) into a **Google BigQuery** data warehouse and is made accessible through **Power BI dashboards**.

## 3. Getting Started with Power BI

### Opening a dashboard

1. Open the **Power BI Desktop** application (or navigate to Power BI Service in your browser).
2. Open the file `SolarPowerProduction.pbit` (Power BI template) or connect to the shared workspace where dashboards have been published.
3. When prompted, sign in with your organisational credentials.
4. Power BI will connect to the **BigQuery** data warehouse (`DataCycle_Warehouse` dataset) and load the latest cached data.

### Refreshing data

Dashboards display data **as of the last scheduled refresh** (see [Section 7](#7-technical-constraints) for refresh timing). To load the most recent data manually:

- In Power BI Desktop: click **Home → Refresh**
- In Power BI Service: click the **Refresh** button on the dataset page

### Common navigation controls

| Control | Action |
|---|---|
| Date slicer / date picker | Filter all visuals to a specific day or date range |
| Dropdown filters | Restrict data to a specific room, inverter, or activity type |
| Cross-filtering | Clicking a chart element (e.g., a bar) automatically filters all other visuals on the page |
| Drill-through | Right-click a data point and select "Drill through" for detailed breakdowns |
| Reset filters | Click the eraser icon or "Reset to default" in the filter pane |

## 4. Dashboards — Director

Directors have access to all strategic dashboards that provide a high-level picture of campus energy flows and room utilisation.

---

### 4.1 Overall Energy Dashboard

**Purpose:** Monitor the campus energy balance — how much solar energy is being produced versus consumed — and track performance trends over time.

#### Key metrics displayed

| Metric | Description | Unit |
|---|---|---|
| Total Solar Production | Energy generated by all solar inverters by end of day | kWh |
| Total Consumption | Total electrical energy consumed by the campus by end of day | kWh |
| Production vs. Consumption Difference | Net energy position (positive = surplus, negative = deficit) | kW |
| % Inverters Running | Proportion of the 5 inverters currently active and producing power | % |

#### Available filters

| Filter | Description |
|---|---|
| **Date range** | Restrict the view to a specific day, week, month, or custom range |
| **Time of day** | Zoom into morning, afternoon, or evening production/consumption windows |

#### How to use this dashboard

- Use the **date range** filter to compare energy production across different periods (e.g., summer vs. winter).
- The **Production vs. Consumption Difference** trend line helps identify days when the campus was self-sufficient (positive values) versus drawing from the grid (negative values).
- The **% Inverters Running** card gives a quick health indicator — a value below 100% during daylight hours warrants attention from the technical team.

> **Tip:** Hover over any data point to see the exact timestamp and value in the tooltip.

---

### 4.2 Rooms Dashboard

**Purpose:** Understand campus room occupancy — which rooms are in use, booking rates over time, and how many rooms are available at any given hour.

#### Key metrics displayed

| Metric | Description | Unit |
|---|---|---|
| % Rooms Booked | Proportion of all campus rooms that have an active reservation | % |
| Rooms Free Count | Absolute number of rooms without a reservation at a given time | Rooms |
| Room occupancy timeline | Hour-by-hour booking status for each room | Schedule view |
| Reservations by activity type | Breakdown of bookings by course type, department, or instructor | Count / % |

#### Available filters

| Filter | Description |
|---|---|
| **Date** | View occupancy for a specific day |
| **Time of day** | Inspect availability during specific hours (e.g., 08:00–12:00) |
| **Room** | Focus on a single room or a group of rooms |
| **Department** | Filter reservations by academic department |
| **Activity type** | Filter by type of booking (lecture, lab, meeting, etc.) |

#### How to use this dashboard

- Use the **date** filter combined with the **time of day** filter to plan maintenance windows in unoccupied rooms.
- The **% Rooms Booked** KPI card gives an instant read of overall campus utilisation for the day.
- The **occupancy timeline** view lets you spot patterns — peak hours, consistently underused rooms, or unusual weekend bookings.
- Filter by **department** to understand which faculties drive the highest room demand.

> **Note:** Booking data is updated **weekly** (see [Section 7](#7-technical-constraints)). The dashboard reflects the most recently loaded booking file, not a live calendar.

## 5. Dashboards — Technician

Technicians have access to two operational dashboards focused on equipment performance and energy optimisation.

---

### 5.1 Solar Production & Inverter Monitoring

**Purpose:** Provide granular, inverter-level visibility into the solar production system — enabling fault detection, performance benchmarking, and maintenance planning.

#### Key metrics displayed

| Metric | Description | Unit |
|---|---|---|
| PAC (AC Power Output) | Real-time AC power delivered to the grid / building by each inverter | W |
| Daysum | Cumulative energy produced by each inverter since midnight | Wh |
| PDC1 / PDC2 | DC input power from each of the two string inputs per inverter | W |
| UDC1 / UDC2 | DC voltage on each string input | V |
| UAC | AC grid voltage | V |
| Inverter Temperature | Operating temperature inside the inverter housing | °C |
| Status | Operational state reported by the inverter | Code |
| Error | Active fault code (displays "No Error" when healthy) | Code / Text |
| % Inverters Running | Proportion of inverters with PAC > 0 at any given time interval | % |

#### Available filters

| Filter | Description |
|---|---|
| **Date range** | Select the period of interest for historical analysis |
| **Time of day** | Focus on specific production windows |
| **Inverter** | Isolate data for one or more of the 5 inverters (IDs 1–5) |
| **Error type** | Filter to time intervals where a specific fault code was active |
| **Status code** | Filter by inverter operational status |

#### How to use this dashboard

- **Fault investigation:** Apply the **Error type** filter to display only intervals with active faults. Cross-reference with the **time of day** and **inverter** filters to identify whether a fault is isolated to one unit or systemic.
- **Performance comparison:** Selecting two inverters side-by-side using the **Inverter** filter reveals if one unit is underperforming relative to its peers under identical conditions.

> **Tip:** A `Daysum` value that stops increasing mid-afternoon while other inverters continue rising is a strong indicator of a fault or shutdown event on that unit.

---

### 5.2 Energy Optimization & Predictions

**Purpose:** Compare actual energy production and consumption against ML-generated forecasts to evaluate system performance and plan operational decisions.

#### Key metrics displayed

| Metric | Description | Unit |
|---|---|---|
| Predicted Production (PAC) | Hourly solar output forecast from the KNIME Random Forest model | W |
| Predicted Production (Daysum) | Total predicted production for the day | Wh |
| Predicted Consumption | Forecast campus energy consumption | kWh |
| Actual vs. Predicted Production | Overlay of measured vs. forecast solar output | W |
| Actual vs. Predicted Consumption | Overlay of measured vs. forecast consumption | kWh |
| Weather conditions | Current temperature (°C), humidity (%), and weather forecast flag | Mixed |

#### Available filters

| Filter | Description |
|---|---|
| **Date** | Select the day for which to view actuals and predictions |
| **Time of day** | Zoom into specific forecast windows |
| **Weather site / measurement** | Filter weather data by measurement point or variable type |

#### How to use this dashboard

- **Daily performance check:** Compare the **Actual vs. Predicted Production** curves. A consistent shortfall against the forecast on a clear day may indicate a soiled panel surface or underperforming inverter.
- **Consumption anomalies:** A significant deviation between actual and predicted consumption can signal unusual equipment usage, HVAC issues, or data quality problems worth investigating.
- **Weather correlation:** Use the **Weather conditions** visuals alongside production data to contextualise deviations — lower-than-predicted output is expected on overcast days; it is only a concern when weather does not explain the gap.
> **Note:** Predictions are generated daily by the KNIME workflow. The solar production model trains on a **7-day rolling window**; the consumption model trains on data from **January 2023 to the present**. Prediction accuracy will naturally be lower during periods with no historical precedent (e.g., first operation after a long shutdown).

## 7. Technical Constraints

Understanding how and when data is updated is important for interpreting dashboard results correctly.

### Data refresh schedule

| Data Type | Frequency | Approximate Availability |
|---|---|---|
| Solar production | Daily | By **~09:15 AM** each morning |
| Power consumption | Daily | By **~09:15 AM** each morning |
| Temperature & humidity | Daily | By **~09:15 AM** each morning |
| Weather forecast | Daily | By **~09:15 AM** each morning |
| Room bookings | Weekly | Updated once per week (Monday) |
| ML predictions | Daily | By **~09:30–10:00 AM** each morning |

The full daily pipeline is orchestrated by **GCP Cloud Workflows** and runs automatically starting at **09:00 AM**. All steps must complete successfully for data to appear in the dashboards.

### Power BI refresh

Power BI dashboards do **not** update in real-time. They display data from the last time the dataset was refreshed against BigQuery. Schedule or trigger a manual refresh after 10:00 AM to ensure you are viewing the latest day's data.

### Data latency

- Data displayed in dashboards is always **at least one day old** (yesterday's data is the most recent).
- The pipeline processes data for the **previous day** (`RUN_DATE = today − 1`).
- Same-day (intraday) monitoring is not supported.

### Data availability window

Historical data is available from **January 2023** onward. Queries or filters set before this date will return no results.

### Missing data days

If the daily pipeline fails (e.g., network issue, VM unavailability), that day's data will not appear in the warehouse. A **backfill** procedure can be run manually by a system administrator to reprocess any missed dates. Contact your administrator if you notice a gap in the time series.

### Known data quality considerations

| Issue | Impact | Mitigation |
|---|---|---|
| Inverter not running (PAC = 0 at night) | Normal — not an error | Night-time zeros are expected; `Pct_Inverters_Running` reflects this |
| Weather sensor failure (value = −99999.0) | Row excluded from warehouse | Gaps in weather data may appear on charts for those intervals |
| Booking fields showing "EMPTY" | Missing metadata in source | Use filters to exclude EMPTY values when summarising by department or instructor |
| European decimal format in source (comma separator) | Handled automatically in pipeline | No user action required |
| Weekly booking data | Dashboard may lag 1–6 days behind actual bookings | Always check the "data as of" date displayed in the Rooms Dashboard header |

## 8. Troubleshooting & FAQ

**Q: The dashboard shows no data for today.**
The pipeline processes yesterday's data and is available from ~09:15 AM. If it is before that time, today's data has not yet been loaded. If it is after 10:00 AM and data is still missing, trigger a manual Power BI dataset refresh. If data is still absent, contact the system administrator to check pipeline logs.

---

**Q: The Rooms Dashboard shows bookings from last week, not this week.**
Booking data is extracted weekly. If the weekly extraction has not yet run, the dashboard will display the previous week's data. The weekly extraction runs on **Monday mornings**.

---

**Q: One inverter shows PAC = 0 throughout the day even in sunlight.**
This likely indicates a fault or shutdown on that inverter. Open the **Solar Production & Inverter Monitoring** dashboard, filter to that specific inverter, and check the **Error** and **Status** fields for that day. Escalate to the maintenance team if an active error code is present.

---

**Q: Predicted and actual production values differ significantly on a sunny day.**
The production model trains on a 7-day rolling window. If the previous week had unusual conditions (grid outage, maintenance shutdown), the model will have learned an atypical baseline. Check the **Inverter Error** data for the training window. The model accuracy will self-correct within a few days of normal operation resuming.

---

**Q: I need data before January 2023.**
Historical data before January 2023 has not been loaded into the warehouse. Contact your system administrator to assess whether historical source files are available for a backfill.

---

**Q: I cannot connect Power BI to BigQuery.**
Ensure your account has been granted `BigQuery Data Viewer` access to the `DataCycle_Warehouse` dataset in the `project-d31bc18d-8d9f-48db-a77` GCP project. Contact your GCP administrator if access is missing.
